3. AI Pipeline — Step by Step

1
Chunk rows into batches
BATCH_SIZE (default 25) · last batch may be smaller · chunk() utility
2
Build prompt for each batch
buildSystemPrompt() — role, 15-field schema, 9 hard rules, few-shot examples
buildUserMessage() — JSON: { columns, rows: [{ rowIndex, values }] }
3
Call LLM with retry + fallback
callWithFallback: primary model → fallback[0] → fallback[1] → ...
Each model: up to MAX_RETRIES (default 2) with exponential backoff (1s, 2s, 4s)
429/5xx → retryable · 401/4xx → fatal, skip to next model
4
Parse LLM JSON response
Strip markdown fences · find outermost JSON block · extract records[] array
Unparseable response → entire batch marked "extraction_failed"
5
Collect across all batches
Accumulate CrmRecord[] + CandidateMeta[] · fire onProgress after each batch
6
postProcess — enforce business rules
trim all whitespace → skip records without email AND mobile → clamp crm_status + data_source to allowed enums (fuzzy-map informal labels) → normalize created_at to ISO 8601 → split multiple emails/phones: first → field, rest → crm_note

4. AI Prompt Rules

R1 Missing data → empty string (never null, "N/A", or omit key)
R2 Never invent data — empty string over guess
R3 crm_status: only GOOD_LEAD_FOLLOW_UP | DID_NOT_CONNECT | BAD_LEAD | SALE_DONE
R4 data_source: only 5 project values (or empty)
R5 created_at → ISO 8601 · parseable by new Date()
R6 Multi email/phone → first in field, rest appended to crm_note
R7 crm_note: catch-all for remarks, extra contacts, comments
R8 No line breaks in values — escape as \n if necessary
R9 Skip rule: no email AND no mobile → skip (missing_contact)

5. Request Lifecycle (SSE Stream)

Client
User clicks "Confirm import"
POST /api/import/stream · multipart/form-data · file field
Server — multipart.ts
readFileField() — extract file bytes from FormData
Handles File, Blob, buffer
Server — runImport()
Size check → parse CSV → classify input → extractLeads()
Error mapping: 400 MISSING_FILE · 413 FILE_TOO_LARGE · 400 INVALID_CSV · 422 EMPTY_CSV
Server — extractLeads()
Chunk rows (BATCH_SIZE=25) → for each batch: build prompt → call LLM → parse JSON
Per-model retry (MAX_RETRIES=2) → fallback model chain
onProgress(batchIndex, totalBatches, processed) after each batch
Client ← SSE
event: progress { batchIndex, totalBatches, processed }
ImportProgress component updates bar + percentage
Server — postProcess()
All collected candidates: trim → skip contactless → clamp enums → normalize dates → split multi-contact
Returns { imported: CrmRecord[], skipped: SkippedRecord[] }
Client ← SSE
event: complete { imported, skipped, stats }
ResultsTable + StatsCards render
